Instruzioni per importare la grammatica OCL in quella ATL: (da usare solo se non parte l'intero progetto)
1. Creare un nuovo workspace
2. Creare un nuovo progetto Xtext e generare xtext artifact dal file MyAtl.xtext
3. Fare il punto 2 per MyOcl
A questo punto, avete due progetti Xtext nel workspace
4. Aprire il file MANIFEST.MF nella sottocartella org.xtext.example.myatl/META-INF(questo nome dipende dal nome dato in fase di creazione)
5. Andare nella sezione dependencies (tab sotto) e mettere nei Required plug-in org.xtext.example.myocl (anche qui, l'ultimo namespace dipende dalla fase di creazione)
6. Fare la stessa cosa del punto 5 nella cartella org.xtext.example.atl.ui e mettere come plug-in org.xtext.example.myocl.ui
7. Infine, andare nella prima cartella del progetto e in particolare nel sotto-package org.xtext.example.atl\src\org.xtext.example.mydsl\ e aprire il file GenerateMyAtl.mwe2
8.A file file troverete il tag language come segue:

language = StandardLanguage {
			name = "org.xtext.example.mydsl.MyAtl"
			fileExtensions = "myatl"	
			
       

			serializer = {
				generateStub = false
			}
			validator = {
				// composedCheck = "org.eclipse.xtext.validation.NamesAreUniqueValidator"
			}
			}

dovete aggiungere il tag  referencedResource =  "platform:/resource/org.xtext.example.myocl/model/generated/MyOcl.genmodel" come segue:

language = StandardLanguage {
			name = "org.xtext.example.mydsl.MyAtl"
			fileExtensions = "myatl"
			
			 referencedResource =
        "platform:/resource/org.xtext.example.myocl/model/generated/MyOcl.genmodel"

			serializer = {
				generateStub = false
			}
			validator = {
				// composedCheck = "org.eclipse.xtext.validation.NamesAreUniqueValidator"
			}
			
			
		}

9. Modificare la prima riga del file MyAtl.xtext come segue:
da grammar org.xtext.example.mydsl.MyAtl with org.eclipse.xtext.common.Terminals 
diventa grammar org.xtext.example.mydsl.MyAtl with org.xtext.example.mydsl.MyOcl
infine aggiungere l'import nello stesso file import "http://www.xtext.org/example/mydsl/MyOcl"

10. Lanciare una nuova applicazione di Eclipse dalla prima cartella di Atl



Note: Ci sono degli override dovuti al fatto che MyOcl usa alcune regole che sono usate anche da ATL ma non dovrebbe essere fonte di errori
